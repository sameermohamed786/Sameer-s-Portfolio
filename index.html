Case Study

C-Hub: End-to-End Product Design for a Maritime Mobile Ecosystem

Figma Product Design User Onboarding Marketplace

My Role

Lead End-to-End Product Designer (Nov 2025 - Jan 2026). Handled user research, storyboarding, user flows, layout architecture, high-fidelity UI prototyping, and interactive mockups.

Project Overview

C-Hub is an all-in-one maritime ecosystem mobile application designed to simplify workflows for seafarers, harbor captains, and shipping logistics crews. It bridges the gap between fragmented port booking processes, local supplier marketplaces, and real-time weather analytics.

Design Process

We followed a structured Double Diamond UI/UX approach:

  • Discover: Conducted interviews with 15 maritime agents to locate shipping scheduling bottlenecks.
  • Define: Mapped core personas (Captain Lars, Vendor Anya) and created onboarding flows.
  • Develop: Designed grid guides and interactive mobile screen wireframes in Figma.
  • Deliver: Designed dynamic custom icons, dark-theme layout variations, and responsive screens ready for front-end engineers.

Challenges

Seafarers face spotty connection profiles at open sea. Standard data-heavy maps and real-time telemetry API structures cause standard mobile interfaces to break, lag, or fail entirely during operations.

Solutions

Designed a connection-adaptive layout. When network bandwidth drops below standard levels, the UI pivots dynamically to high-contrast simple list elements and switches to offline-cached JSON telemetry charts, displaying last known coordinates clearly.

Tools Used

Figma FigJam Adobe Illustrator CSS Variables

B2B Agricultural Commodity Marketplace Platform

Next.js FastAPI PostgreSQL WebSockets

My Role

Full-Stack Developer. Designed relational schemas, created REST APIs in Python using FastAPI, built the responsive dashboard framework in Next.js, and managed DB versioning structures.

Project Overview

This B2B platform connects large-scale agricultural suppliers and commercial food producers. It allows sellers to catalog bulk crops (rice, wheat, corn) and enables buyers to enter real-time, transparent bidding contracts.

Architecture & Features

The backend operates asynchronously using Python's FastAPI, serving data to a React/Next.js frontend. Database schemas are hosted in PostgreSQL, allowing quick querying of bids, contracts, and shipment logs.

Challenges

Real-time bids must propagate to all connected active bidders immediately. Standard polling creates huge server overhead, while database race conditions can allow two bids at the same timestamp to register incorrectly.

Solutions

Implemented dual solutions: WebSockets for push-based live bid distributions, and database-level transaction locks in PostgreSQL ensuring bids are verified and written in sequential atomic blocks.

Tools & Systems Used

Python 3.11 FastAPI PostgreSQL Next.js Git / GitHub Alembic

OMPOI B2B Escrow & Marketplace Platform

FastAPI Next.js PostgreSQL Role-Based Access Control

My Role

Full-Stack Developer (Mar 2026). Designed and implemented the backend APIs, role database structures, schema upgrade files, and frontend dashboard components.

Project Overview

OMPOI is an enterprise B2B platform managing marketplace catalogs, secure escrow transaction contracts, and shipping workflows. It requires strong role separations to verify user permissions at every stage.

Key Implementations

  • Role-Based Access Control: Created custom middleware in FastAPI ensuring Buyers, Sellers, Agents, and Administrators can only trigger verified endpoints.
  • Database Migration Engine: Configured Alembic environments to run automatic SQL schema revisions during platform upgrades.
  • AI Integration: Built API hooks to support automated product description rendering and price prediction models.

Challenges

Executing schema updates in a production database containing active transactions can lock tables, causing site downtime and interrupting ongoing transactions.

Solutions

Structured Alembic scripts to run with transactional blocks and non-blocking index additions. Designed APIs to fail-safely, ensuring live users experience no disruption during backend deployments.

Tools Used

FastAPI Next.js / React Alembic PostgreSQL TypeScript

Integrated Hospital Patient Database Solution

PL/SQL Oracle DB Performance Optimization Triggers

My Role

Database Developer (Aug 2025). Designed the patient table relationship schemes, configured cross-department data sharing flows, and optimized execution speeds.

Project Overview

Designed a secure, relational database system for a major multi-department hospital. The system facilitates immediate access to patient charts, medication timelines, and room occupancy schedules across various floors.

Key Database Implementations

  • PL/SQL Optimization: Created complex stored procedures to automate cross-billing calculations.
  • Data Integrity: Wrote triggers that automatically log auditor edits to medication lists, maintaining strict compliance records.
  • Indices: Built targeted index maps on patient identification numbers to speed up lookups during emergency intakes.

Challenges

Under peak load, concurrent patient requests from the pharmacy and ER slowed retrieval times to over 4 seconds per record, creating bottlenecking issues for hospital staff.

Solutions

Rebuilt legacy nested queries into optimized join patterns and established table-partitioning based on admission dates. These modifications reduced peak load query responses to just 4 milliseconds.

Tools Used

Oracle SQL Developer PL/SQL Packages Execution Planner
